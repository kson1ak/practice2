# Задание 2.4 — Абстрактный класс vs Интерфейс

## Теоретическое напоминание

| Критерий | Абстрактный класс | Интерфейс |
|----------|-------------------|-----------|
| Отношение | «является» (is-a) | «умеет» (can-do) |
| Наследование | Одиночное (`extends`) | Множественное (`implements`) |
| Поля (состояние) | Могут быть любые | Только `static final` |
| Конструктор | Есть | Нет |
| Методы с реализацией | Да | `default`/`static` (с Java 8) |

**Правило выбора:**
- Если есть общие **поля** (состояние) и отношение «является» → **абстрактный класс**.
- Если нужна **множественная реализация** или отношение «умеет» → **интерфейс**.

## Ваша таблица

| Ситуация | Ваш выбор | Обоснование (простым языком) |
|----------|-----------|------------------------------|
| Все животные имеют имя и возраст, но звучат по-разному | Абстрактный класс | У всех животных есть имя и возраст (общие данные), и каждое животное — это конкретное животное (кошка, собака). Поэтому лучше сделать общий класс-родитель. |

| Все транспортные средства имеют скорость и топливо, но двигаются по-разному | Абстрактный класс | У всех транспортных средств есть скорость и топливо (общие данные), и каждое — это конкретный транспорт (машина, самолёт). Лучше сделать общий класс. |
| Робот, посудомоечная машина и телефон — все умеют подключаться к Wi-Fi | Интерфейс | Это просто умение "подключаться к вайфаю". Робот, посудомойка и телефон — совершенно разные вещи, но все они могут это делать. Добавляем им этот навык через интерфейс. |
| Круг, квадрат и треугольник — все фигуры с площадью и периметром, у всех есть цвет | Абстрактный класс | У всех фигур есть цвет (общие данные), и каждая фигура — это конкретная фигура (круг, квадрат). Лучше сделать общий класс. |
